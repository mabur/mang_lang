Manglang

Mang Lang is an experimental minimalistic programming language. Its primary goal is to be a simple and minimal language that is easy to implement and interpret. Mang lang is similar to the Json data format, but with a minimal set of programming primitives added:

  • References to entries in dictionaries
  • Control flow using if and while
  • First class functions

Mang lang is a purely functional and interpreted language. It takes source code written in Mang lang and evaluates it: 

{
  rectangles = (
    {width = 3 height = 1}
    {width = 6 height = 2}
    {width = 3 height = 6}
    {width = 8 height = 4}
  )
  get_area = in {width height} out mul!(width height)
  areas = map!(get_area rectangles)
  total_area = add!areas
  num_rectangles = count!rectangles
  average_area = div!(total_area num_rectangles)
}

When we evaluate the source code above we get the result below: 

{
  rectangles = (
    {width = 3 height = 1}
    {width = 6 height = 2}
    {width = 3 height = 6}
    {width = 8 height = 4}
  )
  get_area = in {width height} out mul!(width height)
  areas = (3 12 18 32)
  total_area = 65
  num_rectangles = 4
  average_area = 16.25
}

Note that both the input and output of the mang lang interpreter is given as mang lang source code!

Design Trade-offs

Mang lang has the following design trade-offs:

  • Minimalistic, instead of feature-rich.
  • Principled, instead of pragmatic.
  • Functional, instead of imperative or object oriented.
  • Eager evaluation, instead of lazy evaluation.
  • Dynamically typed and interpreted, instead of statically typed and compiled.
  • Code as Data and Data as Code. The built-in data structures (lists and dictionaries) are also used to structure the code itself into multiple expressions, lines and variables. Furthermore, both the input data and output data of the Mang Lang interpretor is expressed in Mang Lang itself!

Building Blocks

Manglang only consists of these basic building blocks.

Atomic data like numbers and characters.

Containers holding data like lists and strings and dictionaries.

References to data inside containers.

Control flow like if and while.

Functions and function application which transform data.
